# Entity Relationship Diagram (ERD)

## Smart Lender – Loan Approval Prediction System

### Overview

The Entity Relationship Diagram (ERD) illustrates the database structure of the **Smart Lender – Loan Approval Prediction System**. It identifies the core entities, their attributes, and the relationships between them. The ERD provides a logical representation of how applicant information, loan applications, prediction results, and machine learning models are connected within the system.

---

## Objective

The primary objective of the ERD is to:

* Organize data efficiently.
* Define relationships between different entities.
* Maintain data consistency and integrity.
* Support the machine learning prediction workflow.
* Simplify database design and future development.

---

## Main Entities

### 1. User

Represents the person interacting with the system.

**Examples:**

* Credit Officer
* Applicant

**Attributes**

* User_ID (Primary Key)
* Name
* Email
* Role
* Password

---

### 2. Applicant_Profile

Stores the applicant's personal and financial details.

**Attributes**

* Applicant_ID (Primary Key)
* User_ID (Foreign Key)
* Gender
* Married
* Dependents
* Education
* Self_Employed
* ApplicantIncome
* CoapplicantIncome

---

### 3. Credit_History

Stores the applicant's previous credit information.

**Attributes**

* Credit_ID (Primary Key)
* Applicant_ID (Foreign Key)
* Credit_History
* Existing_Loans
* Credit_Score

---

### 4. Loan_Application

Contains details of each loan request submitted by an applicant.

**Attributes**

* Loan_ID (Primary Key)
* Applicant_ID (Foreign Key)
* LoanAmount
* Loan_Amount_Term
* Property_Area
* Application_Date
* Status

---

### 5. Model

Represents the machine learning model used for prediction.

**Attributes**

* Model_ID (Primary Key)
* Model_Name
* Algorithm
* Version
* Accuracy

**Example**

* Model Name: XGBoost

---

### 6. Prediction_Result

Stores the prediction generated by the machine learning model.

**Attributes**

* Prediction_ID (Primary Key)
* Loan_ID (Foreign Key)
* Model_ID (Foreign Key)
* Prediction
* Confidence_Score
* Prediction_Date

---

# Entity Relationships

### User → Loan_Application

* One user can submit multiple loan applications.
* Relationship: **One-to-Many (1:N)**

---

### Applicant_Profile → Loan_Application

* One applicant profile can be associated with multiple loan applications.
* Relationship: **One-to-Many (1:N)**

---

### Applicant_Profile → Credit_History

* Each applicant has one credit history record.
* Relationship: **One-to-One (1:1)**

---

### Loan_Application → Prediction_Result

* Every loan application generates one prediction result.
* Relationship: **One-to-One (1:1)**

---

### Model → Prediction_Result

* One machine learning model can generate predictions for many loan applications.
* Relationship: **One-to-Many (1:N)**

---

## Database Flow

User
↓
Applicant Profile
↓
Credit History
↓
Loan Application
↓
Machine Learning Model (XGBoost)

| Prediction_Result | Stores loan approval prediction results         |

## Conclusion

The Entity Relationship Diagram provides a clear and organized database design for the Smart Lender Loan Approval Prediction System. It establishes relationships between users, applicant profiles, credit history, loan applications, prediction results, and the machine learning model. This structure supports efficient data management, maintains data integrity, and enables accurate loan approval predictions using the XGBoost algorithm.
